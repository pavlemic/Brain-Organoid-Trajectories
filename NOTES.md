### Theoretical discussion — primary question and analysis pipeline

**Primary question:** Do brain organoids follow the same transcriptional maturation trajectories as the developing human fetal brain — and where do they diverge?

**8-step conceptual pipeline:**
1. **QC both datasets independently** — filter dead cells, empty droplets, doublets (mitochondrial %, gene count thresholds). Critical to separate technical stress from real organoid stress signals.
2. **Normalize and select HVGs** — normalize to 10k counts, log-transform, select ~2–5k highly variable genes. Reduces noise, focuses on informative genes.
3. **Dimensionality reduction** — PCA (30–50 PCs) → UMAP (2D visualization). First visual readout: do organoid and fetal cells overlap or separate?
4. **Cluster and annotate cell types** — Leiden clustering, then label by marker genes (PAX6/SOX2 = progenitors, EOMES = IPs, TBR1/NEUROD2 = excitatory neurons, GAD1/GAD2 = inhibitory neurons, GFAP = astrocytes). Must know *who* cells are before comparing trajectories.
5. **Integrate the two datasets** — Harmony or scVI to remove batch effects while preserving biology. Risk: over-integration erases the real differences you're looking for. Most methodologically sensitive step.
6. **Trajectory inference with Monocle3** — fits principal graph in UMAP space, assigns pseudotime from a designated root (earliest progenitor). Run separately for organoids and fetal brain.
7. **RNA velocity with scVelo** — unspliced/spliced mRNA ratios show where cells are *going*, not just where they are. Velocity arrows on UMAP reveal if organoid progenitors move toward same neuronal states as fetal cells.
8. **Compare trajectories** — (A) topological alignment, (B) same genes driving maturation at same pseudotime, (C) organoid-specific states (stress clusters), (D) neuronal subtype specificity.

**Expected answer:** Nuanced — faithful in some cell types/stages, divergent in others. Bhaduri et al. 2020 found organoid-specific stress clusters and failure to specify neuronal subtypes. This project will confirm, challenge, or extend those findings.

**Attempt 4 (current, untested) — np.fromstring line-by-line with 64 MB buffer:**
- Reads one gene row at a time, parses with `np.fromstring(line, sep='\t')` which is ~10× faster than pandas for raw numeric strings. 64 MB gzip read buffer reduces decompression overhead. Stores only non-zeros immediately.

- For each cell type: take `min(n_cells, target_per_type)` — rare clusters (astrocyte progenitors ~1.4k, stressed progenitors mixed ~1.1k) contribute all their cells; common clusters get downsampled
- Within each cell type: distribute the quota proportionally across samples, so every sample stays represented
- Random seed fixed for reproducibility
- **Where in pipeline:** after off-target cluster removal (Bhaduri 2020 clusters 9, 10, 16), before integration

**Validation checks after subsampling:**
1. Cell-type composition: full vs subset match within ~1% for common types; all rare types present
2. Per-sample counts: every sample represented, proportions preserved
3. Rerun UMAP + Leiden on subset → should recover same clusters with same markers
4. QC distribution overlay (gene count, mito%): should be visually identical
5. Top-10 marker genes per cluster: ≥80% overlap with full-dataset markers
